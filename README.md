# Subtext

**Make your GitHub profile outstanding.** Subtext reads your real repositories, then lets you edit every detail and style every element to generate a complete, striking profile `README.md` — intro, stats, project showcase, tech badges, and a deep set of decorative blocks. Pick a look, tweak the words, copy the markdown, paste it into your profile repo.

Subtext is built on one idea from research into the GitHub-profile tooling space: the decorative-card ecosystem is saturated, but tools that turn your real work into a polished, editable, *legible* profile are not. Subtext leans into substance + style — "best of both."

## Features

- **Reads your real data** — pulls your public profile, repositories, languages, and stars from the GitHub API.
- **Full editing form** — every piece is editable: name, tagline, about, role, location, what you're working on/learning, links, and tech stack. Pre-filled from your data, then yours to change.
- **15 section types** — about/intro, skill icons, tech-stack badges, stats cards, activity graph, project showcase, trophies, social links, dev quote, support/sponsor, profile views + snake, an editable Mermaid mindmap (with sub-branches), a collapsible journey timeline, an editable FAQ, and a footer.
- **Deep styling** — 6 themes, 8 accent colors + custom picker, 5 header styles, 4 badge styles, horizontal/vertical badge layout, and 5 stats-card layouts (including a true grid).
- **Live GitHub-faithful preview** — renders the README the way it'll look on your profile, with a light/dark toggle. Supports alerts, Mermaid, collapsibles, task lists, and tables.
- **Toggle + drag-to-reorder** sections; **pick exactly which repos** appear in your showcase.
- **Copy markdown or download `README.md`** when you're done.

## Running it

No build step. It's a static site.

```bash
# from the repo root
npm install      # optional — only needed for the dev server
npm run dev      # serves src/ at http://localhost:5173
```

Or just open `src/index.html` directly in a browser.

## How it works

Subtext is a single-page static app:

- `src/index.html` — markup and layout
- `src/styles.css` — all styling (design tokens, the editor panel, the GitHub-faithful preview)
- `src/app.js` — data fetching, the markdown generator, the preview renderer, and all editor logic

It calls the public GitHub REST API directly from the browser. No backend, no keys, nothing stored.

## Limitations & roadmap

- **API rate limit.** Anonymous GitHub API requests are capped at 60/hour per IP. For heavy use or private-repo data, the app needs an authenticated token.
- **"Login with GitHub" (OAuth)** is not implemented, and can't be in a purely static file — the OAuth token exchange requires a server-side secret. Adding it means a small serverless function (e.g. a Vercel/Netlify function) alongside this app. The username-lookup flow is designed to stay as the no-friction default; login would be an alternative entry point, not a rewrite.
- **The intro narrative** is composed from repository *metadata* (languages, top projects, stars, activity). A richer version that reads commit history would need an LLM backend.
- **Widget cards** (stats, streak, trophies, activity graph, skill icons, etc.) are generated by third-party services and render once the README is live on GitHub; the in-app preview shows them as labeled placeholders.

## Tech

Vanilla HTML, CSS, and JavaScript. No framework, no dependencies, no build. The only runtime dependencies are Google Fonts and the third-party README-widget services referenced in the generated markdown.
